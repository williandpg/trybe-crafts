<h1><strong>Trybe Crafts</strong></h1>

<h2><strong>Descrição</strong></h2>
<p align="justify">
  Este exercício simula parte do sistema da empresa fictícia Trybe Crafts, voltada para produtos personalizados.
  O objetivo é fortalecer funcionalidades do e-commerce por meio da criação de funções tipadas com TypeScript.
</p>

<h2><strong>Tecnologias Utilizadas</strong></h2>
<ul>
  <li align="justify">
    <a href="https://www.typescriptlang.org/" target="_blank"><strong>TypeScript</strong></a>:
    tipagem estática para reduzir erros, melhorar a legibilidade e tornar as funções mais seguras e fáceis de manter.
  </li>
  <li align="justify">
    <a href="https://nodejs.org/" target="_blank"><strong>Node.js</strong></a>:
    runtime para executar o projeto e os testes em ambiente JavaScript/TypeScript.
  </li>
  <li align="justify">
    <a href="https://jestjs.io/" target="_blank"><strong>Jest</strong></a>:
    framework de testes automatizados usado para validar o comportamento esperado em cada etapa do exercício.
  </li>
  <li align="justify">
    <a href="https://eslint.org/" target="_blank"><strong>ESLint</strong></a>:
    linting para padronização e melhoria contínua da qualidade do código.
  </li>
</ul>

<h2><strong>Estrutura do Projeto</strong></h2>
<p align="justify">A estrutura do projeto é organizada da seguinte forma:</p>

<pre><code>/
├── src/
│   ├── 01-insert.ts
│   ├── 02-insertProducts.ts
│   ├── 03-findById.ts

├── tests/

</code></pre>

<h2><strong>Notas Adicionais</strong></h2>
<ul>
  <li align="justify">
    O exercício aplica conceitos de tipagem como <strong>Type Annotation</strong>, <strong>Type Aliases</strong> e <strong>Union Types</strong> para representar regras e variações de dados.
  </li>
  <li align="justify">
    Cada etapa é validada por <strong>testes automatizados</strong>, garantindo que as funções atendam aos requisitos e evitando regressões.
  </li>
  <li align="justify">
    O foco principal é consolidar boas práticas de desenvolvimento com TypeScript. Isso inclui modelagem de dados, validação de entradas e manipulação de arrays/objetos com tipos bem definidos.
  </li>
</ul>
